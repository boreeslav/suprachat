Bot API SuperMessenger

HTTP и WebSocket API для ботов: сообщения, медиа, inline-кнопки, меню, режим помощника, управление группами и каналами.

Полная документация в репозитории: docs/07-bot-api.md.

Быстрый старт

  1. Создайте бота в мессенджере («Мои боты»).
  2. Сгенерируйте API-токен в настройках бота (GenerateBotToken).
  3. Вызывайте REST или подключите WebSocket с параметрами login и token в query string.
  4. Для личного чата пользователь должен начать диалог с ботом (/start).

Аутентификация

Cookie мессенджера не нужен. Все методы: login (slug бота) + token (API-токен) в query string URL. GET или POST.

На сервере хранится SHA-256 хеш токена. Параметры POST — в JSON-теле, login/token — только в URL.

POST /api/bot-api/sendMessage?login=mybot_bot&token=...
Content-Type: application/json

{"userLogin":"ivan","text":"Привет!"}

REST — режим помощника (assistant)

Пользователь добавляет бота как помощника; выбранные сообщения отправляются боту в открытом виде с командой из меню.

МетодНазначение
GET/POST /api/bot-api/assistantReplyОтвет на сессию помощника (sessionId, text, медиа, кнопки)

Сессии создаются через messenger RPC InvokeBotAssistant в UI; подтверждение ответа — ConfirmAssistantReply.

WebSocket

wss://<host>/ws/bot?login=<bot>&token=<token>

Одна активная сессия на бота: новое подключение закрывает предыдущее.

События от сервера

typeОписание
connectedПодключение, поле botUserId
messageВходящее сообщение, поле update (как в getMessages)
pongОтвет на ping

Команда клиента

{ "action": "ping" }

supra-bot-api.js по умолчанию автопереподключается и догружает пропущенные сообщения через getMessages?afterMessageId=....

JavaScript

<script src="/docs/supra-bot-api.js"></script>
<script>
const api = new SupraBotApi({ login: 'mybot_bot', token: '...' });

api.connectWebSocket({
  onMessage: (m) => console.log(m),
  onSyncComplete: (n) => console.log('Missed:', n),
}, { reconnect: true, syncMissed: true, lastInboxId: null });

await api.sendMessage({ userLogin: 'user', text: 'Hi' });
await api.sendActivity({ chatId: '...', activityType: 'typing', active: true });
await api.setMenu({ menu: { items: [{ text: 'Справка', message: '/help' }] } });
</script>

Ограничения

  • Только незашифрованные исходящие сообщения (без префикса E1:).
  • Inbox на сервере — 1 сутки.
  • Личный чат по userLogin — только если пользователь уже начал диалог с ботом.
  • Inline-кнопки: при нажатии в inbox приходит buttonPress с sourceMessageId, buttonId, action.
  • Активные статусы в открытом чате также приходят в SyncChatPanel (поле activities).
