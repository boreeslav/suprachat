# Основной функционал

## Назначение

**SuperMessenger** — автономный веб-мессенджер на базе логики OtbMessenger с префиксом API **Supra**. Поддерживает личные и групповые чаты, каналы, ботов, файлы, папки чатов, E2EE, PWA и настройки приватности.

## Роли пользователей

| Роль | Возможности |
|------|-------------|
| **User** | Чаты, профиль, приглашения (до 5 активных ссылок), настройки приватности, создание ботов |
| **Admin** | Всё то же + панель `/admin.html`: пользователи, приглашения, смена типа User/Admin, брендинг приложения |
| **Bot** | Системная роль для ботов; взаимодействие через Bot API |

## Аутентификация и регистрация

- **Вход:** `/login.html` → `POST /api/auth/login` (cookie-сессия, 14 дней sliding).
- **Выход:** `POST /api/auth/logout`.
- **Регистрация:** только по одноразовой ссылке `/register/{token}` или `/+{token}`.
  - Админские приглашения (Admin API).
  - Пользовательские приглашения (`/api/profile/invitations`).
- **Уникальный логин** при регистрации (минимум 4 символа).
- После входа — экран **мастер-пароля** (E2EE); может совпадать с паролем учётной записи.

## Чаты

### Типы

| Тип | Описание |
|-----|----------|
| `direct` | Личный чат между двумя пользователями |
| `group` | Закрытая группа с участниками и админами |
| `public_group` | Публичная группа (в т.ч. встраиваемый виджет `MODE_CHAT`) |
| `channel` | Канал с подписчиками и ролями (owner, admin, author, subscriber) |
| `group_branch` | Подчат (ветка) внутри группы, свой slug в URL |

### Сообщения

- Текст, вложения (файлы/изображения через `mc-content` JSON в теле сообщения).
- **Редактирование** и **удаление** своих сообщений.
- **Пересылка** в другие чаты.
- Inline-кнопки в сообщениях ботов (`PressMessageButton`).
- Статусы: `sent`, `read`, `error` (локальная отправка).
- Разделители дат, системные события в группах (переименование, смена аватара, назначение админа).
- Очистка истории (личные — опционально «и у собеседника»).
- Выход из чата / удаление группы при отсутствии участников.

### Список чатов (сайдбар)

- Поиск по чатам (от 2 символов).
- Папки: пользовательские + встроенная **Архив**.
- Свайп влево/вправо между папками (карусель на мобильном).
- Переключение папки по клику с анимацией перелистывания.
- На мобильном (≤680px): чат открывается на весь экран, **без** подсветки в списке после возврата.
- На планшете/десктопе: активный чат подсвечивается в списке.

## Профиль и приватность

### Свой профиль (настройки)

- Имя, email, телефон.
- Фото: выбор **из файла** или **с камеры** (`getUserMedia`).
- Смена пароля и логина.
- Приватность:
  - `searchableByLogin` — поиск/профиль по логину и deep link `/@login`.
  - `searchableByName` — поиск по имени.
  - `allowInvite`, `showOnlineStatus`, `allowWrite` — `everyone` | `contacts`.

### Контент из админки

- **Приветствие после входа** (`loginWelcomeHtml`) — HTML перед экраном мастер-пароля; в тексте можно использовать `{{appName}}` (подставляется из «Название приложения»).
- **О приложении** (`aboutHtml`), **Помощь** (`helpHtml`) — разделы в настройках профиля; поддерживается HTML.
- **Версия** (`appVersion`) и **changelog** (`changelogHtml`) — в «О приложении» версия открывает модальное окно с историей изменений.

### Статус и «О себе»

- **Статус** (`statusText`, до 20 символов): короткая строка под именем в hub настроек и в сайдбаре; у собеседника — в заголовке личного чата (`mc-header-sub`).
- **О себе** (`aboutText`, до 140 символов): отдельное поле в разделе «Аккаунт»; в карточке чужого профиля показывается ниже статуса/онлайна.

### Просмотр чужого профиля

- По ID: `GET /api/profile/{userId}`.
- По логину: `GET /api/profile/by-login/{login}` (с учётом приватности и контактов).
- QR-код ссылки на профиль/группу/канал.

## Группы

- Создание с выбором участников.
- Название, аватар (файл/камера), приглашение по ссылке (`allowJoinByLink`).
- Участники: добавление, удаление, назначение/снятие админа.
- Создатель группы; при выходе создателя — передача создателя админу или случайному участнику.
- Профиль группы: `GetGroupInfo`, `UpdateGroup`, загрузка аватара `POST /api/group/{chatId}/avatar`.
- **Ветки групп** (`group_branch`): подчаты с собственным slug, CRUD и сортировка через messenger RPC.

## Каналы

- Создание канала с уникальным slug (публичная ссылка `/@slug`).
- Роли: owner, admin, author, subscriber.
- Публикация сообщений авторами; подписчики только читают.
- Публичный просмотр без авторизации: `/api/public/channel/{slug}` и сообщения канала.
- Мягкое удаление и восстановление канала.
- Аватар: `POST /api/channel/{chatId}/avatar`.

## Боты

- Создание бота пользователем (`CreateBot`), настройка имени и аватара.
- Генерация API-токена (`GenerateBotToken`); токен хранится на сервере как SHA-256.
- Запуск диалога с ботом (`StartBot`).
- **Bot API** для интеграций: REST `/api/bot-api/*` и WebSocket `/ws/bot` (см. [07-bot-api.md](07-bot-api.md)).
- Режим **ассистента**: inline-меню, сессии, подтверждение ответов (`InvokeBotAssistant`, `ConfirmAssistantReply`).
- Inbox бота на сервере — хранение 1 сутки.
- Пример реализации: `CursorBot/` (Cursor Agent SDK).

## Блокировки

- Блокировка пользователя (`BlockUser`) — запрет личных сообщений.
- Блокировка группы (`BlockGroup`) — скрытие группы из списка.

## Deep links (публичные ссылки)

Формат: `https://host/@{slug}`

| Slug | Тип |
|------|-----|
| Логин (≥4 символов, без `.`) | Профиль пользователя |
| UUID группы | Группа |
| Slug канала | Канал |
| `/{parentId}/{branchSlug}` | Ветка группы |

### Неавторизованный пользователь

- `GET /api/public/profile/{login}` — только если `SearchableByLogin`.
- `GET /api/public/group/{chatId}` — только если `AllowJoinByLink`.
- `GET /api/public/channel/{slug}` — превью канала и сообщений.
- Карточка превью + кнопка «Написать» / «Вступить» / «Подписаться» → `/login.html?returnUrl=...`.

### Авторизованный пользователь

- Открытие профиля, группы, канала или модалки вступления после загрузки приложения.

## Файлы

- `POST /api/files/upload` — загрузка в чат.
- `GET /api/files/{fileId}` — скачивание (только участник чата).
- `GET /api/files/{fileId}/preview`, `/medium` — превью изображений.
- `GET /api/files/avatar/{userId}` — аватар пользователя (anonymous).
- `GET /api/files/group-avatar/{chatId}` — аватар группы (участник).
- `GET /api/files/group-avatar-public/{chatId}` — для публичного превью (если join by link).
- `GET /api/files/channel-public/{fileId}` — файлы канала для анонимного просмотра.

## Realtime

- **SignalR** `/hubs/messenger`, событие `message`, обёртка `BodyTypeName: SupraMessenger`.
- Присутствие: online / idle / offline, видимость по `showOnlineStatus`.
- Доставка новых сообщений, редактирования/удаления, статусов прочтения, новых чатов, активности (печатает…), очистки истории, обновления группы/канала/бота/брендинга.
- При подключении — `SupraSyncHint` (клиент вызывает `RequestSync`).

## PWA и push-уведомления

- Установка как приложение (Android WebAPK, iOS 16.4+ «На экран Домой», desktop).
- Push через Web Push (VAPID); в уведомлении **не** передаётся текст сообщения.
- Глобальные и per-chat настройки уведомлений.
- Service Worker (`sw.js`) работает только в secure context (HTTPS).
- Подробнее: [PWA-УСТАНОВКА.md](PWA-УСТАНОВКА.md).

## Индикатор сети

На аватаре текущего пользователя в сайдбаре:

| Состояние | Отображение |
|-----------|-------------|
| Подключение | Серое «колесо» |
| Подключено | Зелёный кружок |
| Офлайн (исчерпаны попытки) | Серый кружок + всплывающая ошибка по клику |

Транспорт: SignalR → WebSocket → SupraHost ServerChannel (fallback).

## Мобильный UX

| Элемент | Поведение (≤680px) |
|---------|-------------------|
| Модальные окна | На весь экран (`mapp-modal-overlay`) |
| Меню действий | Bottom sheet с подменю и «Назад» |
| Чат | Полноэкранный, `history.back` для возврата |
| Аватар-источник | Bottom sheet (файл / камера) |

## Кеш и офлайн

- **IndexedDB** (`MessengerCache`): сообщения (текст в `bodyEnc`, формат `L1:`), метаданные, превью изображений. Расшифровка только с мастер-паролем; поиск по тексту в кэше отключён.
- При старте — `RequestSync` + reconcile зависших `sending`.
- Очистка кеша: текущий чат / всё (из настроек).

## Темы

- Светлая / тёмная, сохранение в `localStorage` (`mc-theme:app` или `mc-theme:{chatId}`).
- Персональный фон чата (`useThemeChatBg`), настраивается в «Оформление».
- Масштаб UI на мобильных (5 ступеней, см. [04-frontend-reference.md](04-frontend-reference.md)).

## Режимы встраивания клиента

```javascript
// Полное приложение
new Messenger('#chat', Messenger.MODE_APP, { locale: 'ru' });

// Один чат (виджет)
new Messenger('#chat', Messenger.MODE_CHAT, {
  chatId: '...',
  chatName: '...',
  chatType: 'public_group',
});
```

Точка входа standalone: `supra-integration.js` (проверка `/api/auth/me`, boot).
