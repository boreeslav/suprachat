# Установка приложения (PWA) и push-уведомления

SuperMessenger работает как PWA (Progressive Web App). Его можно установить как
обычное приложение на телефон или компьютер и получать push-уведомления о новых
сообщениях, когда приложение свёрнуто или закрыто.

> Важно: установка и уведомления работают только по HTTPS (защищённое соединение).
> На сервере уже настроены домен и TLS — открывайте сайт по адресу `https://...`.

---

## Что такое WebAPK (Android)

Когда вы устанавливаете приложение в Chrome на Android, браузер автоматически
создаёт **WebAPK** — настоящий пакет приложения, который:

- появляется в списке приложений и на главном экране со своей иконкой;
- запускается в отдельном окне без адресной строки браузера;
- умеет показывать системные push-уведомления;
- обновляется автоматически вместе с сайтом.

Для этого ничего вручную собирать не нужно — достаточно «Установить приложение».

---

## Android (Chrome / Edge)

1. Откройте сайт в **Chrome** (или Edge) по адресу `https://...`.
2. Появится предложение **«Установить приложение»** — нажмите его.
   - Если предложения нет: меню браузера (⋮) → **«Установить приложение»** /
     **«Добавить на главный экран»**.
3. Подтвердите установку. Иконка появится на главном экране.
4. Запустите приложение с главного экрана.
5. Включите уведомления: **Настройки → Уведомления → Включить** и разрешите
     показ уведомлений, когда браузер спросит.

---

## iPhone / iPad (Safari, iOS 16.4+)

На iOS push-уведомления работают **только для установленного PWA** (не в обычной
вкладке Safari) и требуют iOS 16.4 или новее.

1. Откройте сайт в **Safari** по адресу `https://...`.
2. Нажмите кнопку **«Поделиться»** (квадрат со стрелкой вверх).
3. Выберите **«На экран Домой»** → **«Добавить»**.
4. Запустите приложение **с экрана Домой** (это обязательно — из Safari
     уведомления не придут).
5. Включите уведомления: **Настройки → Уведомления → Включить** и разрешите
     показ, когда система спросит.

---

## Компьютер (Chrome / Edge, Windows / macOS / Linux)

1. Откройте сайт в Chrome или Edge.
2. В адресной строке справа нажмите иконку установки (монитор со стрелкой) или
     меню (⋮) → **«Установить …»**.
3. Приложение откроется в отдельном окне и появится в меню «Пуск» / Launchpad.
4. Включите уведомления в **Настройки → Уведомления → Включить**.

---

## Как включить и выключить уведомления

Внутри приложения: **Настройки → Уведомления**.

- Первое включение запросит разрешение браузера/системы — нажмите «Разрешить».
- Повторное нажатие предложит выключить уведомления на этом устройстве.
- Уведомления настраиваются отдельно на каждом устройстве.

### Отключение для отдельного чата или группы

В меню чата/группы (значок «⋯» в шапке или долгое нажатие в списке) есть пункт
**«Отключить уведомления»** / **«Включить уведомления»**. Заглушённые чаты не
присылают push, даже когда общие уведомления включены. Эта настройка хранится на
сервере и действует на всех ваших устройствах.

Пункт **Настройки → Уведомления** работает как общий выключатель: при отключении
перестают приходить уведомления из всех чатов.

Если уведомления заблокированы, разрешите их в настройках сайта/приложения:

- Android Chrome: значок 🔒 в адресной строке → «Разрешения» → «Уведомления».
- iOS: «Настройки» системы → найдите приложение → «Уведомления».
- Десктоп: значок 🔒 в адресной строке → «Уведомления» → «Разрешить».

## Приватность: почему в уведомлении нет текста сообщения

Переписка защищена сквозным (end-to-end) шифрованием — **сервер не имеет ключей и
не может прочитать содержимое сообщений**. Поэтому в push-уведомлении показывается
только **от кого** пришло сообщение (имя отправителя для личного чата или название
группы), но **без текста самого сообщения**. Реальное содержимое расшифровывается
уже после открытия приложения на вашем устройстве.

---

## Когда приходят уведомления

- Push приходит, когда у вас **нет активного открытого приложения** (свёрнуто,
  закрыто, экран заблокирован).
- Если приложение открыто и подключено — сообщение приходит мгновенно по
  реальному времени, отдельный push не дублируется.

---

## Решение проблем

- **Нет кнопки «Установить»** — убедитесь, что открыли сайт по HTTPS и используете
  Chrome/Edge (Android/десктоп) или Safari (iOS). Иногда нужно немного попользоваться
  сайтом, чтобы появилось предложение установки.
- **iOS: уведомления не приходят** — приложение должно быть добавлено «На экран
  Домой» и запущено именно оттуда; нужна iOS 16.4+.
- **Уведомления перестали приходить** — переоткройте приложение (подписка
  обновится автоматически) или выключите и снова включите уведомления в настройках.
- **Уведомления заблокированы** — снимите блокировку в разрешениях сайта/приложения
  (см. раздел выше) и включите уведомления заново.
- **После закрытия снова просит мастер-пароль** — закешированный мастер-пароль
  хранится в защищённом хранилище браузера (IndexedDB). На некоторых телефонах
  система очищает его после закрытия приложения. Чтобы этого не происходило:
  **установите приложение** (см. выше) и запускайте именно установленную иконку —
  для установленного PWA браузер выдаёт «постоянное» хранилище и не вычищает
  данные. Также не используйте «очистители памяти», которые удаляют данные сайтов.
  Приложение само запрашивает постоянное хранилище при входе.

## Для администратора / разработчика (кратко)

- Манифест отдаётся динамически: `GET /manifest.webmanifest` — имя и цвета берутся
  из настроек брендинга (`AppAppearanceService`), иконки лежат в `wwwroot/icons/`.
- Push реализован через стандартный **Web Push + VAPID** (без сторонних сервисов):
  - VAPID-ключи генерируются один раз и хранятся в `data/push/vapid.json`;
  - подписки устройств — в `data/push/subscriptions.json`;
  - публичный ключ: `GET /api/push/vapid-public-key`;
  - подписка/отписка: `POST /api/push/subscribe`, `POST /api/push/unsubscribe`.
- Сервер шлёт push офлайн-получателям при отправке сообщения
  (`SupraMessengerController.BroadcastToAllChatParticipantsAsync`), пропуская
  отправителя и тех, у кого есть активное SignalR-подключение.
- Протухшие подписки (HTTP 404/410) автоматически удаляются при отправке.
- Service Worker (`wwwroot/sw.js`) обрабатывает события `push` и `notificationclick`.
