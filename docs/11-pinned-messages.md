# Закреплённые сообщения

Закреп отображается **полоской над лентой сообщений** в открытом чате. На сервере хранятся только **id сообщения и метаданные** — текст клиент берёт из локального кеша (IndexedDB) или дозагружает через `GetMessagesAround`.

---

## Два уровня закрепа

| Scope | Кто создаёт | Видимость | WebSocket |
|-------|-------------|-----------|-----------|
| **`all`** | Модератор чата | Все участники | `SupraMessagePinned` / `SupraMessageUnpinned` всем участникам |
| **`self`** | Любой участник | Только этот пользователь | Событие только его сессиям (синхронизация устройств) |

### Кто может закрепить «для всех»

| Тип чата | Право |
|----------|-------|
| Личный (`direct`) | Любой участник |
| Группа / ветка | Админ или создатель **корневой** группы |
| Канал | Owner или admin |

Если права «для всех» нет, `PinMessage` создаёт **личный** закреп (`scope=self`).

### Открепление

| Ситуация | Действие модератора | Действие обычного участника |
|----------|---------------------|----------------------------|
| Глобальный закреп (`all`) | Снимает для всех | **Скрывает у себя** (запись `hidden` в персональном слое) |
| Личный закреп (`self`) | — | Снимает свой закреп |

При новом глобальном закрепе того же сообщения персональные скрытия/дубликаты сбрасываются.

При **удалении сообщения для всех** все закрепы (глобальные и персональные) для этого id удаляются автоматически.

---

## Несколько закрепов

- Список сортируется по **`seq`** сообщения (хронология в чате).
- В UI — счётчик `1/N`, кнопки вверх/вниз для переключения.
- Тап по полоске (свёрнутый вид) **разворачивает** превью с markdown; повторный тап по панели не сворачивает — только кнопка «×».
- Клик по развёрнутому закрепу (не по ссылке/copy-тегу) прокручивает ленту к сообщению.

## API мессенджера

| MethodName | Параметры | Ответ |
|------------|-----------|-------|
| `GetPinnedMessages` | `chatId` | `{ success, pinned[] }` |
| `PinMessage` | `chatId`, `messageId` | `{ success, scope: "all" \| "self" }` |
| `UnpinMessage` | `chatId`, `messageId` | `{ success }` |

Элемент `pinned[]` (`SupraPinnedMessageDto`):

| Поле | Описание |
|------|----------|
| `messageId` | GUID сообщения |
| `seq` | Порядковый номер в чате (для сортировки) |
| `pinnedAt` | UTC |
| `pinnedBy` | userId закрепившего |
| `scope` | `all` или `self` |

### SyncChatPanel

В ответе `SyncChatPanel` добавлено поле **`pinnedMessages`** — тот же формат, что `GetPinnedMessages`, для начальной отрисовки панели без отдельного запроса.

---

## Realtime

| Событие | Поля |
|---------|------|
| `SupraMessagePinned` | `chatId`, `messageId`, `seq`, `pinnedAt`, `pinnedBy`, `scope` |
| `SupraMessageUnpinned` | `chatId`, `messageId`, `scope` |

`scope=all` рассылается всем участникам чата; `scope=self` — только сессиям пользователя.

---

## Хранение (FileDataStore)

| Сущность | Назначение |
|----------|------------|
| `SupraPinnedMessageRecord` | Глобальный закреп (chatId + messageId) |
| `SupraPinnedMessageUserRecord` | Персональный слой: `state=pinned` (личный закреп) или `state=hidden` (скрытие глобального) |

Реализация: `SupraMessengerService.Pins.cs`.
